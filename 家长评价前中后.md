# 家长点评标签设计方案（最新版）

## 一、整体结构

最终采用“**三阶段＋两层策略池**”：

| 阶段  | 主要评价内容              |
| --- | ------------------- |
| 学习前 | 启动、任务确认、计划、材料和环境准备  |
| 学习中 | 通用自我调节、具体学科策略、检查与订正 |
| 学习后 | 回顾、收尾、复习、预习和自主拓展    |

其中，学习中分为：

1. **通用策略标签**：适用于大多数学习任务；
2. **学科策略标签**：根据当天的数学、语文、英语任务动态出现。

这个版本采纳了最新讨论中的两个核心调整：学习中同时呈现通用策略和具体学科策略；“检查核对、主动订正”归入当前任务的执行过程，而“回顾总结”仍放在学习后。

标签系统继续保留原来的反馈边界：

* 家长端使用清晰、可观察的行为标签；
* 儿童端转化为更短、更积极的“今日闪光”和“下次小挑战”；
* 每天最多展示2个今日闪光和1个下次小挑战；
* 儿童不看到家长原始1—5分，也不看到优秀、一般或高低等级。

---

# 二、学习前：准备与计划

## 2.1 核心标签

| 行为ID                   | 家长点评标签   | 儿童端·今日闪光 | 儿童端·下次小挑战 | 点击解释                            |
| ---------------------- | -------- | -------- | --------- | ------------------------------- |
| `start_initiatively`   | **主动开始** | 主动开始     | 自己先开始     | 知道学习任务后，不需要家长反复催促，能够主动进入学习状态。   |
| `confirm_tasks`        | **确认任务** | 任务清楚     | 先看清任务     | 开始前查看作业要求、任务清单或学习内容，知道今天需要完成什么。 |
| `order_tasks`          | **安排顺序** | 顺序排好     | 先排好顺序     | 根据任务数量、难度或时间，决定先完成什么、后完成什么。     |
| `plan_time`            | **计划时间** | 时间有数     | 先定好时间     | 对开始时间、完成时间或不同任务需要的时间作出简单安排。     |
| `prepare_materials`    | **备齐学具** | 学具备齐     | 先备好学具     | 开始前准备好需要使用的书本、练习册、文具和其他材料。      |
| `organize_environment` | **整理环境** | 环境就位     | 先整理环境     | 整理桌面和学习区域，减少杂物，形成适合学习的空间。       |
| `manage_distractions`  | **管理干扰** | 干扰收好     | 先收好干扰     | 主动处理手机、玩具、电视、无关网页或其他可能影响学习的事物。  |

## 2.2 标签边界

“主动开始”关注的是**是否进入学习状态**；“计划时间”关注的是**是否安排时间**，二者不合并。

“备齐学具”与“整理环境”也应分开：

* 只把书本文具准备好，可以选择“备齐学具”；
* 同时整理了桌面或学习区域，才选择“整理环境”。

---

# 三、学习中：执行、策略与调节

学习中采用三个区域：

1. 通用调节；
2. 学科策略；
3. 完成检查。

---

## 3.1 通用调节标签

| 行为ID                 | 家长点评标签   | 儿童端·今日闪光 | 儿童端·下次小挑战 | 点击解释                            |
| -------------------- | -------- | -------- | --------- | ------------------------------- |
| `sustain_attention`  | **持续专注** | 专注续航     | 再专注一会     | 能在一段时间内持续投入当前任务，减少无关说话、走动或操作。   |
| `recover_attention`  | **分心回归** | 成功回来     | 分心后回来     | 发现自己分心后，能够重新把注意力放回当前学习任务。       |
| `manage_progress`    | **管理进度** | 进度有数     | 看看还剩多少    | 会查看已经完成什么、还剩什么，并适当调整学习速度或任务顺序。  |
| `regulate_emotion`   | **调节情绪** | 情绪稳住     | 先让心情稳住    | 遇到烦躁、错误或困难时，能够暂停、表达、平复情绪并继续学习。  |
| `try_independently`  | **主动尝试** | 先试一试     | 先试一种方法    | 遇到不会的内容时，先根据已有知识、例题或提示尝试一种方法。   |
| `seek_specific_help` | **具体求助** | 求助清楚     | 说清哪里卡住    | 需要帮助时，能够说明自己哪里不懂、尝试过什么以及需要什么帮助。 |

### 选择边界

* 儿童一直没有分心，不需要选择“分心回归”，可以选择“持续专注”。
* 儿童产生情绪但最终继续学习，可以选择“调节情绪”。
* 家长直接告诉答案，不属于“具体求助”或“主动尝试”。
* “具体求助”不是要求儿童完全独立，而是帮助儿童逐渐学会有效利用外部资源。

## 3.2 数学策略标签池

家长选择“数学”任务后，动态显示以下标签：

| 行为ID                      | 家长点评标签   | 儿童端·今日闪光 | 儿童端·下次小挑战 | 点击解释                         |
| ------------------------- | -------- | -------- | --------- | ---------------------------- |
| `math_understand_problem` | **读清题意** | 题意看清     | 先读清题意     | 完整阅读题目，明确题目提供了什么信息、要求解决什么问题。 |
| `math_mark_information`   | **圈画条件** | 条件找全     | 先圈出条件     | 圈出或标记已知条件、问题、单位和重要限制。        |
| `math_show_steps`         | **分步列式** | 步骤清楚     | 一步一步列     | 面对多步骤问题时，把思考和计算过程分开写清楚。      |
| `math_use_visuals`        | **画图辅助** | 图画帮忙     | 试试画图      | 使用线段图、表格、示意图或简单图形帮助理解数量关系。   |
| `math_check_work`         | **主动验算** | 验算完成     | 算完再验算     | 完成计算后，通过重新计算、逆运算、估算等方式检查答案。  |
| `math_review_errors`      | **整理错题** | 错题收好     | 记下这道错题    | 记录典型错误、错误原因和正确方法，供以后复习。      |

### 不使用“避免马虎”

“马虎”容易被理解成人格或态度问题，不能告诉儿童下一步做什么。应转化成：

* 读清题意；
* 圈画条件；
* 分步列式；
* 主动验算。

| 行为ID                             | 家长点评标签   | 儿童端·今日闪光 | 儿童端·下次小挑战 | 点击解释                        |

| `chinese_mark_key_points`        | **圈画重点** | 重点标好     | 圈出重点      | 阅读时圈画关键词、中心句、重要人物、时间或事件。    |
| `chinese_read_in_sections`       | **分段阅读** | 分段读懂     | 一段一段读     | 将较长文章分段阅读，逐段理解主要内容。         |
| `chinese_summarize`              | **概括内容** | 内容说清     | 说说主要内容    | 能用自己的话概括段落、文章或故事的主要内容。      |

“书写清楚”比“字体工整”更合适，因为它关注的是信息是否清晰可辨，而不是把书法水平当作学习态度。

| `english_understand_instruction` | **读清要求** | 要求看清     | 先看清要求     | 先看清题目要求，是朗读、选择、填空、翻译、背诵还是默写。 |
| `english_chunk_memory`           | **分段记忆** | 分段记住     | 分成几段记     | 将单词、句子或课文分成较小部分，逐段进行记忆。      |
| `english_cover_recall`           | **遮盖背诵** | 自己想起来    | 遮住再想想     | 遮住原文，通过回忆说出或写出内容，而不是一直看着重复。  |
| `english_independent_dictation`  | **独立默写** | 独立默写     | 自己先默写     | 不看原文完成单词、句子或短文默写。            |

不使用“认真背诵”“认真默写”，因为“认真”无法直接观察。应当评价儿童使用了什么具体方法。

## 3.5 完成检查标签

这两个行为归入学习中，因为当前任务在完成检查和修改之前，还没有真正结束。

| 行为ID               | 家长点评标签   | 儿童端·今日闪光 | 儿童端·下次小挑战 | 点击解释                               |
| ------------------ | -------- | -------- | --------- | ---------------------------------- |
| `check_completion` | **检查核对** | 检查完成     | 做完再核对     | 完成当前任务后，对照要求检查有没有漏做、漏写、明显错误或未提交内容。 |
| `correct_errors`   | **主动订正** | 自己订正     | 发现问题就改    | 发现错误或得到提示后，能够自己完成修改，并再次确认修改结果。     |

“检查核对”和“主动订正”不能完全合并：

* 检查核对是发现问题；
* 主动订正是修改问题。

# 四、学习后：回顾、收尾与延伸

## 4.1 学习后标签

| 行为ID                 | 家长点评标签   | 儿童端·今日闪光 | 儿童端·下次小挑战 | 点击解释                               | 使用性质 |
| -------------------- | -------- | -------- | --------- | ---------------------------------- | ---- |
| `reflect_learning`   | **回顾总结** | 今日回顾     | 想想今天的收获   | 学习结束后，回想完成了什么、哪里困难、什么方法有效、下次可以怎样做。 | 核心   |
| `organize_materials` | **整理学具** | 学具归位     | 做完收整齐     | 收好书本、练习册、文具和其他材料，并简单整理学习区域。        | 常规   |
| `review_learning`    | **复习巩固** | 复习完成     | 再复习一个重点   | 通过回忆、复述、朗读、练习或重看重点巩固已经学过的内容。       | 条件   |
| `preview_learning`   | **预习新知** | 预习完成     | 先看看新内容    | 提前浏览接下来要学习的内容，并标记已经理解或暂时不明白的地方。    | 条件   |
| `extend_learning`    | **课外拓展** | 主动探索     | —         | 完成基本任务后，出于兴趣主动阅读、练习、查找资料或探索相关问题。   | 额外闪光 |

## 4.2 使用规则

### 回顾总结

保留在学习后，不移入学习中。

它与检查核对的区别是：

* 检查核对：有没有完成、遗漏或出错；
* 回顾总结：用了什么方法、学到了什么、下次怎样改进。

### 复习巩固和预习新知

只在当天存在相应安排时显示。不选择不能被视为表现不足，也不应自动降低阶段评分。

### 课外拓展

只建议作为“今日闪光”，不设置为“下次小挑战”。

原因是课外拓展属于额外、自主的学习行为，不应给儿童造成“完成基本任务仍然不够”的压力。

# 五、家长端的标签展示结构

## 学习前

默认显示全部7项：

> 主动开始｜确认任务｜安排顺序｜计划时间｜备齐学具｜整理环境｜管理干扰

## 学习中

先显示通用标签：

> 持续专注｜分心回归｜管理进度｜调节情绪｜主动尝试｜具体求助｜检查核对｜主动订正

再根据当天任务显示学科标签：

> 数学策略／语文策略／英语策略

建议家长先选择当天观察到的学科，再展开对应标签，避免一次出现二十多个选项。

## 学习后

默认显示：

> 回顾总结｜整理学具

按当天安排显示：

> 复习巩固｜预习新知

“课外拓展”放在“更多闪光”区域，只能作为肯定标签。

---

# 六、标签选择规则

## 今天值得肯定

* 整个评价最多选择2项；
* 可以跨阶段选择；
* 可以选择1个通用行为＋1个学科策略；
* 允许不选，不能虚构儿童没有出现的行为。

例如：

> 今日闪光：[持续专注] [主动验算]

## 下一步继续练习

* 整个评价最多选择1项；
* 必须是下一次可以直接执行的行为；
* 不强制家长每天选择；
* 课外拓展不进入小挑战；
* “无法观察”和“没有相关任务”不生成小挑战。

例如：

> 下次小挑战：[做完再核对]

这一规则延续了现有方案中“0—2个今日闪光、0—1个下次小挑战”的低负担结构。

---

# 七、点击解释的统一模板

每个标签点击后统一显示三个部分。

## 1. 这是什么意思

用一句话说明行为定义。

## 2. 可以观察到什么

列出2—3个具体表现。

## 3. 什么情况下不要选

说明容易混淆或不适用的情形。

例如：

### 管理进度

**这是什么意思**
学习过程中知道自己完成了什么、还剩什么，并适当调整学习速度。

**可以观察到**

* 主动查看任务清单；
* 知道还有几项任务；
* 发现时间不足时调整先后顺序。

**不要选择**

* 完全由家长不断提醒剩余任务；
* 只是一味加快速度，却没有关注完成质量。

# 八、最终标签数量

| 阶段  | 固定核心标签 |     条件或学科标签 |
| --- | -----: | ----------: |
| 学习前 |      7 |           0 |
| 学习中 |      8 | 数学6、语文7、英语6 |
| 学习后 |      2 |  复习、预习、拓展3项 |

标签库数量较多，但单次评价中实际呈现的数量受到控制：

* 学习前显示7项；
* 学习中先显示8项通用标签，再按学科展开4—7项；
* 学习后通常显示2—4项；
* 家长最终只选择最多2个闪光和1个挑战。

这样既保留了学科策略的具体性，也不会让家长一次面对全部标签。

## 最终冻结版本

### 学习前

**主动开始、确认任务、安排顺序、计划时间、备齐学具、整理环境、管理干扰**

### 学习中·通用

**持续专注、分心回归、管理进度、调节情绪、主动尝试、具体求助、检查核对、主动订正**

### 学习中·数学

**读清题意、圈画条件、分步列式、画图辅助、主动验算、整理错题**

### 学习中·语文

**读清要求、圈画重点、分段阅读、概括内容、联系前文、书写清楚、朗读检查**

### 学习中·英语

**读清要求、分段记忆、遮盖背诵、独立默写、核对拼写、朗读检查**

### 学习后

**回顾总结、整理学具、复习巩固、预习新知、课外拓展**

这一版本可以直接作为家长端标签配置、儿童端映射文案和后续点击解释页面的基础。
