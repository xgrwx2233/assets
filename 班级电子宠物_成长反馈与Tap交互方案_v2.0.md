# 班级电子宠物成长反馈与 Tap 交互方案
**v2.0｜最终整合版**
状态：研究方案与产品交互共同冻结，可进入原型与程序联调。
适用范围：家长每日评价产生的宠物成长反馈；教师结果评价可按相同奖励队列扩展；结构化反思对话不属于本方案。
## 一、版本结论
本版本正式将反馈主流程从“逐标签长语料库”升级为“Tap 领取成长体验”。395条旧语料库不再作为运行时主反馈，但保留归档，供研究过程、标签措辞和未来活动文案参考。
- 特效负责爽感；
- 成长能量数字负责明确进度；
- “今日闪光”标签负责确认具体自主学习行为；
- “下次小挑战”负责提供低压力、可尝试的下一步方向；
- 少量通用宠物短句负责情绪、关系感和陪伴；
- 基础宠物组与结构化反思组使用完全相同的反馈体验。

## 二、旧方案与新方案的关系
| 项目 | 旧版 v1.0 | 新版 v2.0 |
| --- | --- | --- |
| 主要反馈载体 | 395条逐标签宠物语句 | 动画、特效、成长数字、标签与少量通用短句共同完成 |
| 具体行为反馈 | 宠物重复描述行为 | “今日闪光”标签直接呈现具体行为 |
| 改进方向 | 宠物逐条说练习语 | 仅在“今日成长记录”中显示“下次小挑战” |
| 爽感 | 主要依靠文字 | Tap动作、粒子、数字跳动、成长条和标签爆出 |
| 宠物角色 | 像会换句式的评价机器人 | 负责开心、惊喜、期待和陪伴 |
| 运行时语料量 | 395条 | 44条通用短句 |
| 旧语料处理 | 正式调用 | 归档，不删除，停止主流程调用 |

## 三、设计目标与理论功能分工
| 组件 | 儿童看到什么 | 主要功能 |
| --- | --- | --- |
| 宠物动作与视觉特效 | 跳跃、转圈、发光、星星或能量粒子 | 提供即时反馈和游戏爽感 |
| 成长能量数字 | 成长能量 +N | 提供明确的成长进度与胜任体验 |
| 今日闪光 | 0—2个具体行为标签 | 确认今天实际出现的自主学习行为 |
| 下次小挑战 | 0—1个下一步行为 | 提供低压力、非命令的可尝试方向 |
| 宠物情绪短句 | 开心、惊喜、期待或陪伴 | 建立宠物关系感，不承担说教或策略教学 |
| 今日成长记录 | 可反复查看的奖励卡 | 帮助儿童回看当日成长，不展示原始评价等级 |

## 四、不可改变的研究边界
- 特效负责爽感，不让长文字承担全部反馈功能。
- 成长能量显示游戏数值，不显示家长或教师原始评分。
- 第一次Tap只展示奖励与今日闪光，不展示下次小挑战。
- 下次小挑战只在今日成长记录中出现，不扣分、不削弱领奖体验。
- 前台不显示高分、中分、低分、优秀、良好或红黄绿等级。
- 宠物短句只表达情绪和陪伴，不重复解释标签，不承担策略教学。
- 基础宠物组与结构化反思组使用完全相同的反馈流程。
- 结构化反思对话独立存在，不加分、不改变反馈特效。
- 前台不显示家长或教师的原始1—5分；
- 不显示“优秀、良好、一般、高分、中分、低分”或红黄绿等级；
- 不公开排名，不扣除已获得成长，不让宠物生病、哭泣、失望或离开；
- 家长体验两题不进入儿童反馈，不向儿童展示；
- 评分只在后台影响成长能量和内部语气，不成为儿童可见的评价。

| 内部/家长端概念 | 儿童端显示 |
| --- | --- |
| 成长值或转换后的奖励数值 | 成长能量 |
| 今天值得肯定 | 今日闪光 |
| 下一步可以继续练习 | 下次小挑战 |

## 六、反馈输入与输出
### 6.1 输入
- 成长算法生成的 `growthEnergyDelta`；
- 成长能量占当日可获得上限的比例 `growthEnergyRatio`；
- 家长选择的0—2个肯定行为ID；
- 家长选择的0—1个练习行为ID；
- 当前宠物名 `{petName}`；
- 可选成长事件：普通吸收、升级、进化、技能解锁。

### 6.2 输出
- 一次性领奖动画；
- “成长能量 +N”数字特效；
- 0—2个“今日闪光”儿童标签；
- 1条通用宠物情绪短句；
- 可反复打开的“今日成长记录”；
- 记录中的0—1个“下次小挑战”。

## 七、成长能量显示规则
家长原始评分与教师原始评分只用于后台计算。儿童看到的是经过成长算法转换后的游戏数值。
前台统一显示：**成长能量 +{growthEnergyDelta}**。
- 允许显示具体数字，以保留游戏奖励的确定感和爽感；
- 不使用“+5分”“今日得分”等字样；
- 不向儿童展示来自家长还是教师；
- 不拆分家长分、教师分和阶段分；
- 成长算法可在独立配置中调整，问卷与儿童文案不硬编码原始评分。

## 八、完整状态机
| 状态 | 含义 | 主要交互 |
| --- | --- | --- |
| idle_no_reward | 当前没有待领取的新成长 | Tap触发普通宠物互动 |
| reward_pending | 已有新成长，尚未领取 | 宠物周围出现星星、光圈或轻微待领取动作 |
| reward_revealing | 正在播放一次性领奖流程 | 暂不重复触发完整领奖 |
| reward_claimed | 奖励已领取 | 第二次Tap可打开今日成长记录；普通Tap不再重播领奖 |
| growth_record_open | 今日成长记录已打开 | 查看成长能量、今日闪光和下次小挑战 |
| ordinary_interaction | 普通宠物互动播放中 | 完成后回到reward_claimed或idle状态 |

## 九、新奖励到达时的待领取提示
- 宠物周围出现少量小星星；
- 使用柔和光圈或呼吸光；
- 宠物可以有轻微期待动作；
- 可使用小礼物角标，但不得遮挡宠物；
- 提示应让学生想点击，但不能使用红色警告、倒计时或损失暗示；
- 不得使用悲伤、生病、停止成长或“快来救宠物”的压力设计。

## 十、第一次 Tap：完整领奖闭环
第一次Tap不是只打开一个弹窗，而是在约2.2—3.0秒内自动完成“动作—数字—标签—情绪”的连续反馈。无需第三次Tap才能听到宠物说话。
| 时间 | 步骤 | 呈现内容 | 实现要点 |
| --- | --- | --- | --- |
| 0—0.3秒 | 宠物即时反应 | 跳一下、转圈、挥手或身体发光 | 点击后必须立刻有响应；无专用动画时用轻微弹跳与光效回退 |
| 0.3—0.8秒 | 成长能量爆出 | “成长能量 +N” | 数字轻微放大，粒子飞入成长条，成长条同步增加 |
| 0.7—1.5秒 | 今日闪光出现 | 0—2个儿童标签 | 标签依次弹出或上浮，只显示肯定，不显示小挑战 |
| 1.5—2.6秒 | 宠物情绪短句 | 随机1条celebrate / warm / support短句 | 短句只表达情绪和陪伴，不重复解释标签 |
| 2.2—3.0秒 | 收束 | 标签轻轻飞入成长记录 | 宠物回到可交互状态，完整领奖仅播放一次 |

### 10.1 第一次Tap的显示优先级
- 成长能量数字必须显示；
- 有肯定标签时，最多显示2个“今日闪光”；
- 没有肯定标签时，不虚构行为，只显示成长能量与通用情绪短句；
- “下次小挑战”绝不进入第一次领奖爆发画面；
- 升级、进化或技能解锁时，可在通用情绪短句之后追加独立成长事件气泡。

### 10.2 爽感原则
- 点击后300毫秒内必须有动作或光效响应；
- 数字变化、成长条变化和粒子方向应一致；
- 标签最多2个，避免信息过载；
- 一条完整领奖动画只播放一次，避免刷动画；
- 允许跳过动画，但跳过后仍需立即完成能量入账和记录保存；
- 支持“减少动态效果”，减少粒子和位移，但保留数字、标签与文字。

## 十一、第二次 Tap：今日成长记录
奖励领取完成后，第二次Tap宠物或点击独立的成长记录入口，打开可反复查看的奖励卡。
推荐界面：
**今日成长**

⚡ 成长能量 +12

✨ 今日闪光
[主动开始] [成功回到任务]

🌱 下次小挑战
[看看有没有漏掉]
### 11.1 页面结构
| 区块 | 数量 | 规则 |
| --- | --- | --- |
| 成长能量 | 1项 | 显示当次或当日汇总的成长能量，不显示原始评分 |
| 今日闪光 | 0—2项 | 使用儿童短标签；无标签时可隐藏整个区块 |
| 下次小挑战 | 0—1项 | 只作为可尝试方向，不扣分、不影响已获得成长 |
| 成长事件 | 0—1项 | 升级、进化或技能解锁时显示 |

### 11.2 禁止显示
- 家长原始1—5分
- 教师原始评分
- 阶段评分
- 高/中/低
- 优秀/良好/一般
- 家长姓名或评价来源

## 十二、后续普通 Tap
- 领取完成后，再次Tap宠物只播放普通互动动作和1条普通互动短句；
- 不重复播放完整领奖动画；
- 不重复弹出成长能量数字；
- “今日成长记录”仍可通过第二次Tap或独立按钮反复打开；
- 普通互动建议设置约1.8秒冷却，避免高速连点造成声音和气泡堆叠；
- 最近3条普通互动语不优先重复。

## 十三、内部语气池
内部保留三种语气，但儿童前台绝不显示对应等级。
| 内部名称 | 适用条件 | 情绪方向 | 禁止误解 |
| --- | --- | --- | --- |
| celebrate｜庆祝 | 高成长能量、2个今日闪光、升级/进化/技能解锁 | 兴奋、惊喜、庆祝 | 不是“高分等级” |
| warm｜温暖 | 正常成长、1个今日闪光、稳定完成 | 开心、满足、期待 | 不是“中分等级” |
| support｜陪伴 | 成长能量较少、无肯定标签、只有小挑战或观察信息有限 | 温暖、轻松、继续向前 | 不是“低分台词”，不能暗示失败 |

### 13.1 语气选择优先级
1. 存在level_up、evolution或skill_unlocked成长事件 → `celebrate`
2. growthEnergyRatio >= 0.75 或今日闪光标签数量 >= 2 → `celebrate`
3. growthEnergyRatio >= 0.40 或今日闪光标签数量 = 1 → `warm`
4. 其余有效成长记录、仅有下次小挑战或观察信息有限 → `support`
support语气禁止使用“没关系”“今天表现不好”“今天辛苦了”等会让儿童猜测自己被低分评价的表达。
## 十四、家长标签到儿童标签的映射
家长端保留专业、清晰的行为描述；儿童端使用更短、更游戏化的标签。同一个行为ID连接两端，数据库保存ID而不是重复保存中文。

2. 今天先休息一下，{petName}也来充充电！
3. 轻松的一天也很珍贵，{petName}好开心！
**观察信息不足或无具体闪光标签**
1. 今天也留下小脚印啦，{petName}好开心！
2. 新的成长还在路上，{petName}好期待！
3. 今天也一起向前走，{petName}陪着主人！
## 十六、宠物短句语言边界
- 默认不使用“你”，需要称呼时使用“主人”；
- 不使用“我”作为宠物自称，统一使用 `{petName}`；
- 一条气泡只有一个短句；
- 主要表达开心、惊喜、兴奋、期待、满足或陪伴；
- 不重复朗读“今日闪光”和“下次小挑战”的完整标签；
- 不出现家长、教师或评分来源；
- 不评价聪明、天赋、人格、学霸或全班排名；
- 不使用命令、羞辱、比较、失望、威胁和情感控制；
- 不把宠物成长与儿童服从绑定，不说“不努力宠物就不长大”。

## 十七、程序事件数据契约
每次待领取成长应形成一个不可重复领取的 `rewardEvent`。推荐结构：
```json
{
  "rewardEventId": "reward_2026-07-15_student-001_parent",
  "date": "2026-07-15",
  "source": "parent_daily",
  "configVersion": "2.0.0",
  "status": "pending",
  "growthEnergyDelta": 12,
  "growthEnergyRatio": 0.8,
  "praiseBehaviorIds": [
    "start_initiatively",
    "recover_from_distraction"
  ],
  "challengeBehaviorId": "check_omissions",
  "tone": "celebrate",
  "growthEvent": null,
  "createdAt": "2026-07-15T20:00:00+08:00",
  "claimedAt": null
}
```
### 17.1 字段说明
| 字段 | 作用 |
| --- | --- |
| rewardEventId | 唯一标识；用于防止重复领取和重复播放完整动画 |
| source | parent_daily、teacher_result或system；儿童前台不显示来源 |
| growthEnergyDelta | 儿童可见的游戏成长能量增量 |
| growthEnergyRatio | 内部语气选择使用，不在儿童前台显示 |
| praiseBehaviorIds | 0—2个今日闪光行为ID |
| challengeBehaviorId | 0—1个下次小挑战行为ID |
| tone | celebrate、warm或support，仅内部使用 |
| growthEvent | 升级、进化或技能解锁等可选事件 |
| status / claimedAt | 记录待领取、播放中和已领取状态 |

## 十八、与家长评价、教师评价和反思对话的边界
### 18.1 家长每日评价
- 三阶段评分进入成长算法；
- “今天值得肯定”ID转为儿童端“今日闪光”；
- “下一步可以继续练习”ID转为“下次小挑战”；
- 两道家长体验题不参与成长、不进入儿童端。

### 18.2 教师结果评价
- 教师结果评价可生成独立的成长事件并进入同一奖励队列；
- 不得显示教师原始评分；
- 本版本不擅自定义教师行为标签，待教师评价方案最终冻结后另行扩展；
- 若同一天家长与教师事件同时到达，可由成长引擎合并为一次领取，或按事件队列依次领取，但同一事件不得重复。

### 18.3 结构化反思对话
- 反思对话只存在于反思组；
- 不增加成长能量；
- 不触发额外Tap特效；
- 不解锁额外奖励；
- 不使用本方案的通用情绪语承担策略教学；
- 基础组与反思组的成长反馈体验必须完全一致。

## 十九、无任务、观察不足与空标签处理
| 情形 | 处理 |
| --- | --- |
| 今天没有相关学习任务 | 不扣成长；不显示虚构的今日闪光；可使用轻松休息类特殊短句 |
| 家长基本未观察 | 不因缺少观察而扣成长；不虚构行为；使用中性陪伴短句 |
| 有评分但未选择肯定标签 | 显示成长能量与通用情绪短句；“今日闪光”区块隐藏 |
| 只选择了下次小挑战 | 第一次Tap不显示小挑战；在今日成长记录中显示；情绪使用support |
| 成长能量为0 | 不显示“+0”；转为无任务或无有效成长记录的中性状态，由业务规则决定 |

## 二十、视觉、音效与可访问性
- 动作、粒子、数字和成长条要同步，不要各自无关地播放；
- 特效以宠物为中心，避免全屏强闪；
- 每次只突出一个数字和最多两个标签；
- 标签不能只靠颜色区分，应同时有图标和文字；
- 音效建议0.3—0.7秒，支持静音；
- 提供减少动态效果：降低粒子、位移和缩放幅度，但保留信息；
- 完整领奖流程允许跳过，跳过不影响领取结果；
- 避免频繁震屏、强烈闪白和持续高音。

## 二十六、最终冻结结论
正式反馈体验冻结为：
> **一次Tap领取成长：宠物动作＋成长能量＋今日闪光＋宠物情绪短句；第二次Tap查看今日成长记录，其中包含今日闪光与下次小挑战；后续Tap恢复普通宠物互动。**
具体行为由结构化标签呈现，宠物语言只承担情绪和陪伴；成长数字是游戏能量而不是原始评分；“下次小挑战”不进入领奖爆发画面，也不削弱已获得的成长。
